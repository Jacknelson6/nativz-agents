# Project Management — Expert Knowledge Base

## Content Production Pipeline

### Pipeline Stages
```
Brief → Script/Concept → Shoot → Edit → Internal Review → Client Review → Revisions → Final Approval → Schedule → Publish
```

### Stage Details & Timelines

**1. Brief (Day 1)**
- Account Manager creates the content brief based on strategy, client goals, and performance data
- Brief includes: objective, platform, format, target audience, key message, CTA, reference examples, brand guidelines, due date
- **Template fields**: Client, Campaign, Content Type, Hook Angle, Key Talking Points, Visual Direction, Audio/Music Notes, Caption Direction, Hashtags, Publish Date

**2. Script/Concept (Days 1-2)**
- For scripted content: full script with hook, body, CTA
- For UGC/creator content: creator brief with talking points (not full script — needs to feel natural)
- For graphic content: copy + layout mockup
- **Approval gate**: Client approves concept/script before production begins (prevents wasted production)

**3. Shoot (Days 2-4)**
- In-house filming or creator filming
- Shoot multiple versions/hooks when possible (2-3 hook variations per piece)
- Capture B-roll while on set — always more than you think you'll need
- **Deliverable**: Raw footage uploaded to shared drive within 24 hours of shoot

**4. Edit (Days 3-6)**
- Editor receives footage + brief + approved script
- First cut delivered within 2-3 business days
- Include captions, text overlays, music, color grading, platform-specific formatting
- Export in all required platform formats (TikTok 9:16, IG Reels 9:16, YouTube 16:9 if applicable)

**5. Internal Review (Day 6)**
- Account Manager + Creative Director review before client sees it
- Check: brand consistency, brief alignment, technical quality, caption accuracy, platform specs
- Catch issues internally — never send half-baked work to the client

**6. Client Review (Days 6-8)**
- Send via shared folder or project management tool with clear feedback deadline
- Include: video file, caption, hashtags, posting date, platform
- **Set expectation**: "Please provide feedback by [date]. If no feedback received, we'll proceed with publishing."

**7. Revisions (Days 8-9)**
- Max **2 rounds of revisions** included in standard scope (define this in the contract)
- Minor revisions (copy tweaks, timing adjustments): 24-hour turnaround
- Major revisions (re-edit, new footage needed): 2-3 business day turnaround
- Track revision requests to identify clients who consistently over-revise (scope creep signal)

**8. Final Approval → Schedule → Publish (Days 9-10)**
- Client gives written approval (email or Monday.com sign-off)
- Content scheduled via native platform tools or scheduling tool (Later, Sprout Social, etc.)
- Post goes live on approved date/time

### Pipeline Timing Summary
| Stage | Duration | Owner |
|---|---|---|
| Brief creation | Day 1 | Account Manager |
| Concept/script | Days 1-2 | Strategist / AM |
| Client concept approval | Days 2-3 | Client |
| Shoot | Days 3-4 | Production / Creator |
| Edit | Days 4-6 | Editor |
| Internal review | Day 6 | AM + Creative Director |
| Client review | Days 6-8 | Client |
| Revisions | Days 8-9 | Editor |
| Final approval + schedule | Day 9-10 | Client → AM |

**Total: ~10 business days from brief to publish.** Build this into content calendars — if a post needs to go live March 15, the brief must be created by March 1.

## Monday.com Workflow Best Practices

### Board Structure
Create one board per client with these groups:
- **Backlog** — ideas and upcoming briefs
- **In Production** — active items moving through pipeline
- **In Review** — waiting for internal or client feedback
- **Approved** — ready to schedule/publish
- **Published** — archive of completed content
- **On Hold** — paused items

### Essential Columns
| Column | Type | Purpose |
|---|---|---|
| Status | Status | Pipeline stage (Brief, Script, Shoot, Edit, Review, Approved, Published) |
| Owner | People | Who's responsible for current stage |
| Due Date | Date | When this stage's deliverable is due |
| Publish Date | Date | When content goes live |
| Platform | Dropdown | TikTok, IG, YouTube, etc. |
| Content Type | Dropdown | UGC, Product Demo, Tutorial, Testimonial, etc. |
| Priority | Priority | High / Medium / Low |
| Files | Files | Attach briefs, scripts, footage, final assets |
| Client Approval | Status | Pending / Approved / Changes Requested |
| Link | Link | Link to live post after publishing |

### Automations to Set Up
- When **Status** changes to "In Review" → notify Account Manager
- When **Status** changes to "Client Review" → notify client (if they're on Monday.com) or trigger email
- When **Due Date** is today and **Status** isn't complete → notify Owner
- When **Client Approval** changes to "Approved" → move to Approved group
- When **Status** changes to "Published" → move to Published group

### Dashboard Views
- **My Work**: Filter by Owner = you, sorted by Due Date
- **Client Calendar**: Calendar view by Publish Date, filtered by client
- **Pipeline Overview**: Status overview showing count of items per stage
- **Overdue Items**: Filter Due Date < today and Status ≠ Published

## Deadline Management & Buffer Strategies

### Buffer Rules
- **Always build in 2 business days of buffer** between "content ready" and "publish date"
- **Client feedback buffer**: Assume clients will take the full time you give them (and sometimes more). If you need feedback by Wednesday, tell them Tuesday.
- **Revision buffer**: Always assume at least 1 round of revisions. Schedule the first draft delivery 3 days before the real deadline.

### Managing Late Content
When content is running behind:
1. **Identify the bottleneck** — is it production, client feedback, or revisions?
2. **Communicate immediately** — tell the client if a publish date is at risk (before it's missed, not after)
3. **Offer solutions**: Swap publish dates with another piece, simplify the content scope, extend the deadline
4. **Prevent recurrence**: Track delays by cause. If clients consistently delay approvals, implement a "silence = approval" policy after 48 hours.

### Rush Requests
For urgent/unplanned content:
- **Rush turnaround**: 2-3 business days (vs standard 10)
- **Requires**: Dropping or delaying other work — be transparent about trade-offs
- **Rush fee**: If frequent, build rush fees into the contract (25-50% surcharge)
- **Template content**: Maintain a library of quick-turn templates (quote graphics, simple text overlays, repurposed clips) for reactive moments

## Resource Allocation Across Clients

### Capacity Planning
- **Each editor** can handle 15-25 pieces of short-form content per week (depending on complexity)
- **Each account manager** can effectively manage 5-8 clients (depending on scope)
- **Each strategist/media buyer** can manage 4-6 ad accounts actively

### Allocation Principles
1. **Revenue-proportional** — clients paying more get proportionally more resources (but set minimums for all clients)
2. **Growth-proportional** — allocate extra resources to clients with scaling potential or retention risk
3. **Batch by client** — editors work better focusing on one client per day rather than switching between 5
4. **Balance the calendar** — don't stack all deadlines on Fridays. Spread publish dates across the week.

### When Capacity is Maxed
Signals you're at capacity:
- Deadlines slipping consistently
- Quality of internal reviews declining
- Team working overtime regularly
- Buffer time disappearing

Solutions (in order):
1. Improve efficiency (templates, automations, batch workflows)
2. Reduce scope per client (fewer pieces, simpler formats)
3. Hire contractors/freelancers for overflow
4. Hire full-time (if demand is sustained 3+ months)
5. Pause new client acquisition until capacity stabilizes

## Quality Control Checkpoints

### Pre-Delivery QC Checklist
Before any content goes to the client:
- [ ] Matches the brief (message, CTA, tone, platform)
- [ ] Correct aspect ratio and resolution for target platform
- [ ] Text is readable on mobile (test by viewing on phone at arm's length)
- [ ] Audio levels are balanced (voice clear, music not overpowering)
- [ ] Captions are accurate (no spelling errors, brand names correct)
- [ ] Brand guidelines followed (colors, fonts, logo usage)
- [ ] Safe zones respected (no text/key elements cut off by platform UI)
- [ ] No copyrighted music (if content will be used as an ad)
- [ ] Caption/hashtags included and proofread
- [ ] Correct publish date assigned
- [ ] All platform versions exported (TikTok, IG, YouTube, etc.)

### Post-Publish QC
Within 2 hours of publishing:
- [ ] Post is live and displaying correctly
- [ ] Links work (link in bio updated, UTMs correct)
- [ ] Captions and hashtags posted correctly
- [ ] Tagged accounts are correct
- [ ] No comments that need immediate response (spam, questions, complaints)

## Handoff Protocols Between Teams

### Account Manager → Editor Handoff
**What the editor needs:**
- Completed brief (no gaps — editors shouldn't have to guess)
- Raw footage (organized, labeled, in shared drive)
- Approved script/concept
- Reference examples (links to content in the style you want)
- Platform and format specs
- Deadline (with buffer built in)
- Any client-specific notes (brand quirks, things to avoid)

### Editor → Account Manager Handoff
**What the AM needs from the editor:**
- Final exports (all platform versions)
- Project file location (for future revisions)
- Notes on any creative decisions that deviated from brief (and why)
- Any concerns about quality or client expectations

### Account Manager → Media Buyer Handoff
**When passing creative to ads team:**
- Final ad creative files (correct specs for each platform)
- Campaign objective and target audience
- Key messaging and CTA
- Landing page URL (confirmed working)
- Budget parameters
- Launch date and flight dates
- Any creative testing plan (which variations to test)

### Handoff Rules
1. **Written, not verbal** — all handoffs happen in Monday.com or a shared doc, never just a conversation
2. **Complete information** — the receiving person should be able to do their job without asking clarifying questions
3. **Confirmation** — the receiving person acknowledges the handoff and confirms timeline
4. **Single source of truth** — all project files, feedback, and approvals live in one place (Monday.com + shared drive)
5. **No "toss over the wall"** — if you're handing off, you're still accountable for the outcome. Check in after handoff to ensure nothing was lost.
